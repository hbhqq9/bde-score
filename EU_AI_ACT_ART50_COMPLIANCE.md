# EU AI Act Article 50 - Transparency Compliance Statement
## BDE Score - AI Agent Reliability Assessment

**Effective Date**: 2026-08-01 (before Art.50 enforcement on 2026-08-02)
**Last Updated**: 2026-07-12
**Applicable Services**: BDE Score API, MCP Server, Landing Page

---

## 1. Scope & Applicability

BDE Score provides AI Agent reliability assessment services under the EU AI Act. This document addresses Article 50 transparency obligations.

### Risk Classification
- **BDE Score System**: General Purpose AI (not classified as High-Risk under Annex III)
- **Assessed Systems**: Third-party AI Agents (user-deployed, outside BDE Score control)
- **Obligation Level**: Article 50 transparency requirements apply

---

## 2. Article 50(1) - AI System Disclosure

**Requirement**: Providers shall ensure that AI systems are identifiable as AI systems when interacting with humans.

### BDE Score Implementation

#### 2.1 Service Identification
- All API responses include `X-BDE-AI-System: true` header
- Landing Page displays "Powered by BDE Score AI Assessment Engine"
- MCP Server tool responses include AI system identification in metadata
- Dashboard reports clearly label all outputs as "AI-Generated Assessment"

#### 2.2 Assessment Output Labeling
Every BDE Score assessment output contains:
```json
{
  "ai_system_disclosure": {
    "is_ai_generated": true,
    "provider": "BDE Score",
    "model_type": "rule-based + LLM analysis",
    "confidence_range": "0-100 (BDE Score)",
    "limitations": "Assessment based on publicly accessible signals; not a comprehensive security audit"
  }
}
```

---

## 3. Article 50(2) - Content Disclosure

**Requirement**: Users shall be informed when content is AI-generated.

### BDE Score Implementation

#### 3.1 Report Generation
- HTML compliance reports display: "This assessment was generated by BDE Score AI Engine"
- JSON API responses include `assessment_methodology: "ai-automated"` field
- Compliance check results explicitly state scoring methodology
- All generated content includes version timestamp and methodology reference

#### 3.2 User Notifications
- First-time users see methodology explanation on Landing Page
- API documentation discloses AI-based scoring approach
- Error messages distinguish between AI assessment errors and system errors

---

## 4. Article 50(3) - Synthetic Content Disclosure

**Requirement**: Deployers of AI systems generating synthetic content shall disclose the AI-generated nature.

### BDE Score Implementation

BDE Score does **not** generate synthetic text, images, audio, or video. 
This obligation does not apply to BDE Score's core functionality.

**Clarification**: BDE Score outputs are structured assessments (JSON/HTML), not synthetic media.

---

## 5. Article 50(4) - Deep Fake Disclosure

**Requirement**: Providers of AI systems generating deep fakes shall disclose this.

### BDE Score Implementation

BDE Score does **not** generate, manipulate, or process deep fake content.
This obligation does not apply.

```json
  }

```

### 6.3 Landing Page Disclosure
The Landing Page includes:
- Clear statement that assessments are AI-generated
- Methodology summary
- Limitations disclosure
- Link to full Privacy Policy and Terms of Service

## 7. Record Keeping

### 7.1 Assessment Logs
- All assessments logged with timestamp, input URL, output score
- Logs retained for 12 months per GDPR Article 30 requirements
- Logs include AI system version and methodology reference

### 7.2 Audit Trail
- Each assessment produces a unique assessment ID
- Assessment IDs are traceable to methodology version
- Change log maintained for scoring algorithm updates

- How to improve their score
- Data processing details (see Privacy Policy)

- Users can request manual review of automated assessments
